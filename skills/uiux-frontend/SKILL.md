# UIUX Frontend Skill

## 使う場面
- LP/コーポレートサイトの導線改善
- セクション再配置・文言改善
- 可読性/アクセシビリティ改善

## 標準ワークフロー
1. 目的を定義（例: CV率向上、問い合わせ到達率向上）。
2. 対象ページを特定（`index.html` / `corporate.html` / `lp.html` / `ec.html`）。
3. 変更は小さく分けて実装。
4. 共通ヘッダー/フッターの崩れを横断確認。
5. 375px/1440pxで表示確認。

## 実装チェック
- 見出し階層が連続している（h1→h2→h3）。
- CTAがファーストビューかその直下にある。
- リンク文言が具体的である。
- 画像のaltが用途に応じて適切。
- キーボード操作で主要導線に到達可能。
- フォント・カラー規約（CLAUDE.md）を遵守。

## 出力フォーマット
- 変更目的
- 変更ファイル
- 期待効果
- 確認コマンド
